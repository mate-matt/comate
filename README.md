# Codex Mate

[中文说明](./README.zh-CN.md)

Codex Mate is a fully local companion gallery for the Codex desktop app. It indexes images generated by Codex Desktop, links them with local session metadata and prompts, and gives you a clean private gallery for browsing your work.

![Codex Mate screenshot](./docs/images/codex-mate-screenshot.png)

## Why

Codex Desktop can generate useful images across many sessions. Codex Mate makes those local images easier to review, search, and open without uploading anything or depending on a cloud service.

## Privacy

- Fully local.
- No login, API key, analytics, or telemetry.
- No network connection is required for your image library.
- Reads only local Codex Desktop data on your machine.
- Stores its own local SQLite index under `~/.codex-mate` by default.

## Codex Desktop Data

Codex Mate is built for the Codex desktop app, not Codex CLI. By default it reads:

```text
~/.codex/generated_images
~/.codex/session_index.jsonl
~/.codex/sessions
```

You can override paths with environment variables:

```bash
CODEX_HOME=/path/to/codex-data
CODEX_MATE_HOME=/path/to/codex-mate-data
CODEX_MATE_DB=/path/to/codex-mate.sqlite
```

## Install

```bash
npm install
```

## Development

Run the web UI and API in development mode:

```bash
npm run dev
```

Default development URLs:

- Web UI: `http://127.0.0.1:4388`
- API: `http://127.0.0.1:4389`

## Desktop App

Run the Electron desktop app:

```bash
npm run desktop
```

Build unsigned macOS artifacts:

```bash
npm run package:mac
```

Artifacts are written to `release/`.

## Production Web Mode

```bash
npm run build
npm start
```

## Test

```bash
npm test
npm run build
```

## GitHub Actions

This repository includes:

- `CI`: runs tests and production builds on push and pull requests.
- `Package macOS App`: manually packages unsigned macOS `.dmg` and `.zip` artifacts, and creates a draft release for `v*` tags.

## macOS Gatekeeper

Current public builds are unsigned. If macOS blocks launch, right-click the app and choose **Open**, or allow it from **System Settings > Privacy & Security**.

Signed and notarized releases require Apple Developer credentials and are not configured by default.
